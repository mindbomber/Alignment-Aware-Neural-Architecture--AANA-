Pre-tool-call contract

Paste a tool call. AANA checks before execution.

An agent proposes a tool call. AANA checks evidence, authorization, and risk. The tool only executes if the route is accept.

Pick an example, click Run Gate, then inspect the route and execution proof. A plain permissive agent would execute the proposed tool call; AANA blocks unless the contract is satisfied.

  • accept allows execution
  • ask, defer, or refuse blocks execution
  • missing auth/evidence becomes a blocker
  • runtime recommendation can be overridden
  • audit-safe decision data is emitted
